Module 3 - Guide 14

One student creates a repository with a few files and invites all the other students as collaborators to the repository

The students will then receive an email where they need to accept the collaboration The students will then need to do the following:

Clone the repository to their own computer Make a new branch in the recently cloned repository Make sure that you are working on the new branch Make some changes to the code Switch to main branch Observe that your changes are not there anymore Switch back to your original branch and observe the changes comming back Push the new code to GitHub by writing "git push origin xxxxx" where xxxxx is the name of your branch Observe the new code in GitHub Make a pull request (on GitHub) with your new branch DO NOT MERGE THE PULL REQUEST

When the other people in your group have done that as well you can do the following:

Check out pull requests in GitHub Choose one pull request and write a comment to the pull request (code review) Merge the pull request into main Now you should be able to see all the changes on the main branch in your repository. other students can now switch to the main branch and pull the code to get the newest changes
